# React 18 vs 19 — кратко для команды

Проект на **React 19**. Ниже — что изменилось по сравнению с 18 и как это влияет на то, **как мы пишем код**.

---

## Главное: React 19 ≠ «хуки кеширования больше не нужны»

`useMemo`, `useCallback` и `memo` **остались**. Они по-прежнему решают две задачи:

1. **Дорогие вычисления** — не пересчитывать тяжёлую логику на каждый рендер.
2. **Стабильные ссылки** — когда дочерний компонент или `useEffect` завязан на ссылку/объект в зависимостях.

В React 19 появился **React Compiler** (опционально): он может *автоматически* мемоизировать там, где это безопасно. Но:

- компилятор **не включён по умолчанию** во всех проектах;
- он не отменяет необходимость понимать, *зачем* вы мемоизируете;
- для **примитивов** (`number`, `string`) и **простых операций** `useMemo` чаще всего **лишний**.

### Пример из `DonutChart`

```ts
// Достаточно — массив легенды маленький, reduce дешёвый
const total = chartData.reduce((sum, item) => sum + item.value, 0);
```

```ts
// useMemo здесь избыточен: накладные расходы ≈ самому reduce
const total = useMemo(
  () => chartData.reduce((sum, item) => sum + item.value, 0),
  [chartData],
);
```

**Когда `useMemo` уместен:** фильтрация тысяч строк, тяжёлый парсинг, построение дерева, агрегации для графиков на больших данных.

## Что нового в React 19 (практика)

| Тема | React 18 | React 19 |
|------|----------|----------|
| **Actions** | async в `useEffect` + ручной loading/error | `useActionState`, формы с `action`, меньше boilerplate |
| **Optimistic UI** | вручную через Redux / локальный state | `useOptimistic` |
| **Ref на компонент** | `forwardRef` обязателен | `ref` как обычный prop (меньше обёрток) |
| **Context** | `Provider` + `value` | `<Context value={…}>` — `Provider` не обязателен |
| **Метаданные документа** | react-helmet и аналоги | встроенные `<title>`, `<meta>` в компонентах |
| **Hydration errors** | часто «тихие» расхождения | более явные сообщения |
| **Strict Mode** | двойной mount в dev | поведение уточнено, dev по-прежнему строже prod |

## Как писать код в нашем проекте (рекомендации)

### 1. Не мемоизируйте «на автомате»

```ts
// ❌ Лишнее
const label = useMemo(() => user.name, [user.name]);

// ✅ Достаточно
const label = user.name;
```

### 2. Мемоизируйте осознанно

- тяжёлый расчёт;
- объект/массив/функция, передаётся в `memo`-компонент или в deps `useEffect`;
- измеренная проблема производительности  

### 3. D3 / canvas / сторонние DOM

Как в `useDonutChartSvg`: побочный эффект в `useEffect` — нормальный паттерн и в 18, и в 19. React 19 это не заменяет.

### 4. Redux Toolkit

RTK уже мемоизирует селекторы (`createSelector`). Дублировать `useMemo` вокруг `useSelector` без причины не нужно.

### 5. Стили (styled-components)

Создание `styled.div` **вне** тела компонента (как в `ui/`) — правильно. Внутри render — плохо в любой версии React.

## React Compiler — включён в наших apps

**Почему не было «из коробки»:** React 19 — это библиотека. Compiler — отдельный **Babel-плагин** в цепочке сборки (Vite). Его сознательно не включают автоматически: другой build, возможны edge cases, команда подключает сама.

**Что сделано в monorepo:**

```bash
npm install -D babel-plugin-react-compiler   # корень, devDependency
```

В `apps/app-*/vite.config.ts`:

```ts
react({
  babel: {
    plugins: ['babel-plugin-react-compiler'],
  },
}),
```

Работает с `@vitejs/plugin-react@5` (у нас сейчас v5). На v6+ inline `babel` убрали — тогда нужен `@rolldown/plugin-babel`.

**Как писать код:** без лишних `useMemo` / `useCallback` / `memo` — компилятор мемоизирует сам, где безопасно.

**Проверка:** React DevTools → у оптимизированных компонентов бейдж **Memo ✨**.

**Если компонент ломается после включения:**

```ts
function Problematic() {
  'use no memo';
  // ...
}
```

**Не покрывается компилятором:** `core` собирается через `tsc` (без Babel) — только код в `apps/*/src` через Vite.

## Шпаргалка: нужен ли хук?

| Ситуация | Нужен `useMemo`? |
|----------|------------------|
| `a + b`, `array.length`, простой `reduce` на <100 элементах | Нет |
| Сортировка/фильтр 10k+ записей | Да |
| Объект `{ a, b }` в props `memo`-дочернего | Да (или убрать `memo` у ребёнка) |
| Значение только в JSX как число/строка | Нет |
| Зависимость `useEffect` — новый объект каждый рендер | Да `useMemo` / стабилизировать данные |
