# Локализация

В проекте есть два типа переводов:

- `common` — общие тексты из `@org/core`: кнопки, переключатели, навигация.
- `incidents`, `users`, etc. — тексты конкретных модулей.

Главная идея простая: **core даёт провайдер и общие переводы, модуль даёт только свои словари**.

## Самая Важная Схема

```text
apps/app-operator/src/app/providers.tsx
  -> AppI18nProvider modules={appI18nModules}
      -> core/i18n/instance.ts
          -> подключает common из core
          -> подключает словари модулей из appI18nModules
```

У модуля нет своего `I18nProvider`. Модуль просто экспортирует объект с переводами.

## Где Провайдер

Провайдер подключается один раз на всё приложение:

```tsx
// apps/app-operator/src/app/providers.tsx

<AppI18nProvider modules={appI18nModules}>
  <AppThemeProvider>{children}</AppThemeProvider>
</AppI18nProvider>
```

`AppI18nProvider` лежит в core:

```text
core/i18n/AppI18nProvider.tsx
```

Он делает три вещи:

- запускает `i18next`;
- хранит текущий язык;
- принимает словари модулей через `modules`.

## Где Переводы Core

Core-переводы лежат здесь:

```text
core/i18n/locales/ru/common.ts
core/i18n/locales/en/common.ts
```

Они собираются здесь:

```text
core/i18n/coreResources.ts
```

И подключаются автоматически внутри:

```text
core/i18n/instance.ts
```

Поэтому `common` не нужно руками добавлять в `appI18nModules`.

## Где Переводы Модуля

Например, модуль `incidents` хранит свои переводы рядом с собой:

```text
apps/app-operator/src/modules/incidents/locales/
  ru.ts
  en.ts
  index.ts
```

В `ru.ts` и `en.ts` лежат сами тексты.

В `index.ts` они упаковываются в namespace модуля:

```ts
export const incidentsI18nResources = {
  incidents: {
    ru: incidentsRu,
    en: incidentsEn,
  },
};
```

Название `incidents` — это namespace. Потом в компоненте мы пишем:

```tsx
const { t } = useTranslation('incidents');
```

## Где Модули Подключаются К Приложению

Все словари модулей собираются в одном файле:

```text
apps/app-operator/src/app/i18n/modules.ts
```

Пример:

```ts
export const appI18nModules = [
  incidentsI18nResources,
];
```

Дальше этот массив передаётся в `AppI18nProvider`:

```tsx
<AppI18nProvider modules={appI18nModules}>
```

## Как Использовать Переводы

Для общих core-текстов:

```tsx
import { useTranslation } from '@org/core';

const { t } = useTranslation('common');

t('locale.toggle');
```

Для текстов модуля:

```tsx
import { useTranslation } from '@org/core';

const { t } = useTranslation('incidents');

t('page.title');
```

Важно: `useTranslation` берём из `@org/core`, а не напрямую из `react-i18next`.

## Как Добавить Новый Модуль

Допустим, появился модуль `users`.

1. Создать папку:

```text
apps/app-operator/src/modules/users/locales/
```

2. Добавить файлы:

```text
ru.ts
en.ts
index.ts
```

3. В `index.ts` экспортировать ресурсы:

```ts
import type { ModuleI18nResources } from '@org/core';

import { usersEn } from './en';
import { usersRu } from './ru';

export const usersI18nResources: ModuleI18nResources = {
  users: {
    ru: usersRu,
    en: usersEn,
  },
};
```

4. Подключить модуль в приложении:

```ts
// apps/app-operator/src/app/i18n/modules.ts

import { incidentsI18nResources } from '@/modules/incidents/locales';
import { usersI18nResources } from '@/modules/users/locales';

export const appI18nModules = [
  incidentsI18nResources,
  usersI18nResources,
];
```

5. Использовать в компоненте:

```tsx
const { t } = useTranslation('users');

t('page.title');
```

## Ошибки В Redux

В Redux лучше хранить не готовый текст, а ключ перевода:

```ts
state.error = 'errors.loadFallback';
```

А переводить уже в UI:

```tsx
{error ? <Error>{t(error)}</Error> : null}
```

Так язык можно переключить без перезаписи store.

## Коротко

```text
core/i18n/AppI18nProvider.tsx
  провайдер i18n

core/i18n/locales/*/common.ts
  общие переводы core

modules/incidents/locales/*
  переводы модуля incidents

app/i18n/modules.ts
  список словарей модулей

app/providers.tsx
  место, где всё подключается
```
